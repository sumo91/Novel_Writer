## V3 Memory And Indexing Friction

This manual run exposed concrete friction:

- Acceptance packet quality depends heavily on manual editing; generated packets need stricter V2.5 templates.
- Review JSON allows invalid dimension values in early chapters, even when total score passes. A validator should enforce 0-10 dimension scores.
- Pending approvals repeat across chapters and need dedupe with source chapter metadata.
- Drift review required reading many separate files manually; a report generator should aggregate chapter index, reviews, timeline, open threads, and change log.
- Chapter 4 exposed arc outline drift; long-form runs need automatic warnings when the current arc lacks future chapter goals.
- Accepted chapter state can contain stale wording if acceptance packet text was written before acceptance; accept workflow should check for `ready_for_acceptance` or `pending human acceptance` residue.
